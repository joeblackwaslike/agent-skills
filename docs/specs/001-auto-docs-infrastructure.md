# Spec: Auto-Docs Infrastructure

## Problem

`skills/working-with-claude-code/references/` contains 126 markdown files generated by a fetch script. There is no standard way to run that script, no convention for future skills that need the same pattern, and no CI to keep docs fresh.

## Deliverables

### 1. Root `Makefile`

Auto-discovers update scripts by globbing `skills/*/scripts/update*.{js,sh}`. Targets:

| Target | Description |
|---|---|
| `make update-all` | Run every discovered update script |
| `make update-<skill-name>` | Run one skill's script explicitly |
| `make list-update-scripts` | Print discovered scripts |

Discovery uses `find skills -path '*/scripts/update*.js' -o -path '*/scripts/update*.sh'` so new skills with a script at `skills/<name>/scripts/update.js` are picked up automatically.

### 2. `.github/workflows/update-docs.yml`

- Schedule: weekly, Monday 04:00 UTC (`cron: '0 4 * * 1'`)
- Trigger: `workflow_dispatch` for manual runs
- Steps: checkout → setup Node 20 → `make update-all` → `git add skills/*/references/` → commit + push if dirty (skips commit if nothing changed)
- No `npm install` needed — `update_docs.js` uses only Node stdlib

### 3. CLAUDE.md addition

New section documenting the convention (see below).

---

## Convention: Skills with Auto-Generated Docs

Any skill that wraps external documentation follows this contract:

- **Script location:** `skills/<name>/scripts/update.js` (or `update.sh`)
- **Output location:** `skills/<name>/references/`
- **Exit behavior:** exit non-zero on failure; exit 0 on success
- **Docs are content:** generated files are committed to the repo, not gitignored
- **Discovery:** the root Makefile and CI workflow discover scripts automatically via glob — no registration needed

Existing example: `skills/working-with-claude-code/scripts/update_docs.js`
