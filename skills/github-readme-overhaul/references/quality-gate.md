# README Quality Gate

Run this before claiming the README is done.

## First Screen

- Product name is obvious.
- Tagline states pain and outcome.
- Badge row is accurate and not noisy.
- Hero proof explains the product.
- Infographics communicate the main claims without requiring long paragraphs.
- Install or primary action is visible.
- Reader can tell who this is for.

## Truthfulness

- No fake metrics.
- No unsupported performance claims.
- No invented testimonials.
- No screenshots that imply features not implemented.
- Roadmap is separated from current features.

## GitHub Rendering

- Relative links resolve.
- Images live in the repo and use correct paths.
- Terminal demo GIF/video paths resolve and are legible at README width.
- Tables are readable on mobile.
- Mermaid diagrams render or have fallback.
- HTML is minimal and GitHub-safe.
- Dark/light contrast is acceptable.

## Conversion

- Install command is copy-pastable.
- Quickstart reaches first value quickly.
- Example output is concrete.
- Sections are visually scannable, not walls of text.
- Configuration is not introduced before first success unless required.
- CTA is useful, not needy.

## Maintenance

- Badges point at active workflows/packages.
- Screenshots have stable paths.
- Generated assets are clearly named.
- README does not depend on external images likely to rot.
- Contribution and license sections match repo reality.

## Final Report

State:

- files changed
- badges verified
- install command checked or not checked
- visual assets added or still needed
- terminal demo recorded, scripted, or still needed
- any claims that need owner confirmation
