# Working with Git

Offline reference for **Git the version-control system**, pinned to a known Git release (see [`PINNED_VERSION`](PINNED_VERSION) / [`references/VERSIONS.md`](references/VERSIONS.md), resolved via Homebrew). Command and concept pages in [`references/cli/`](references/cli/) are the **AsciiDoc sources from `git/git`** at the pinned tag; the full **Pro Git** book (2nd ed.) is in [`references/book/`](references/book/); opinionated how-to guides are in [`references/howto/`](references/howto/). Fetched files carry `source`/`fetched_at`/`sha256` frontmatter; the how-to guides are hand-written.

## How to use this skill

1. **Exact command behavior/flags** — read `references/cli/git-<command>.adoc`. These are the literal man-page sources for the **pinned** Git version; trust your local `git help <command>` if it disagrees and surface the drift.
2. **Conceptual understanding** — read the relevant Pro Git chapter in `references/book/` (filenames are the chapter path flattened with `__`).
3. **"How do I safely do X"** — start with `references/howto/*.md`; these are the opinionated playbooks for the dangerous/confusing operations.
4. **Searching** — `grep -ri <term> references/` across all three.

## Behavioral guidance

- **History rewriting is destructive.** On shared branches, always `git push --force-with-lease` (never bare `--force`), and confirm no one else has the branch. See `references/howto/rewriting-history.md`.
- **`git stash` drops untracked files** unless you pass `-u`. Reach for `git stash -u` when in doubt.
- **Reflog is the safety net** — almost any "lost" commit is recoverable via `git reflog` for ~90 days. Check it before declaring data lost.
