# devcontainer CLI Reference

```bash
npm install -g @devcontainers/cli
devcontainer --version
```

---

## Core Commands

### `devcontainer up`

Create and start the dev container.

```bash
devcontainer up --workspace-folder .
devcontainer up --workspace-folder /path/to/project
devcontainer up --workspace-folder . --remove-existing-container   # force recreate
devcontainer up --workspace-folder . --id-label project=myapp      # label for identification
```

**Key flags:**
- `--workspace-folder` — path to project with devcontainer.json (required)
- `--config-path` — explicit path to devcontainer.json (multi-config projects)
- `--remove-existing-container` — destroy and recreate
- `--mount` — additional bind mount: `"type=bind,source=/host,target=/container"`
- `--update-remote-user-uid-default` — auto-sync UID (never | on | default)

---

### `devcontainer build`

Build (and optionally push) the dev container image for pre-building.

```bash
devcontainer build --workspace-folder .
devcontainer build --workspace-folder . --image-name ghcr.io/org/mydev:latest
devcontainer build --workspace-folder . --image-name ghcr.io/org/mydev:latest --push true
devcontainer build --workspace-folder . --cache-from ghcr.io/org/mydev:cache
```

**Key flags:**
- `--image-name` — tag for the built image
- `--push` — push to registry after build (`true`/`false`)
- `--cache-from` — image(s) to use as build cache

Pre-building is the recommended workflow for Codespaces and teams: build once in CI, everyone pulls the pre-built image.

---

### `devcontainer exec`

Execute a command inside a running container with full devcontainer context (lifecycle env, remoteUser, remoteEnv).

```bash
devcontainer exec --workspace-folder . npm test
devcontainer exec --workspace-folder . bash -c "echo hello"
devcontainer exec --workspace-folder . --remote-user root apt-get update
```

**Key flags:**
- `--remote-user` — override `remoteUser` from config
- Applies `remoteEnv` and `userEnvProbe` settings

---

### `devcontainer read-configuration`

Parse and output the resolved devcontainer.json configuration (useful for debugging).

```bash
devcontainer read-configuration --workspace-folder .
devcontainer read-configuration --workspace-folder . | jq '.configuration.features'
```

Use this to verify the config is valid before running `up`.

---

### `devcontainer features test`

Run the feature test suite against local or published features.

```bash
# Test a local feature
devcontainer features test -f ./src/my-feature

# Test against multiple base images
devcontainer features test -f ./src/my-feature --base-image ubuntu:24.04
devcontainer features test -f ./src/my-feature --base-image alpine:latest

# Test a specific scenario
devcontainer features test -f ./src/my-feature --scenario install_with_extra

# Test all features in the repo
devcontainer features test

# Skip auto-generated tests, run scenarios only
devcontainer features test -f ./src/my-feature --skip-autogenerated

# Test a published feature
devcontainer features test --features ghcr.io/devcontainers/features/node:1
```

---

### `devcontainer templates apply`

Apply a template to a project directory.

```bash
devcontainer templates apply \
  -t ghcr.io/devcontainers/templates/python:latest \
  -w /path/to/project

# With options
devcontainer templates apply \
  -t ghcr.io/devcontainers/templates/python:latest \
  -a '{"imageVariant": "3.12"}' \
  -w .
```

---

### `devcontainer features publish` / `templates publish`

Publish features or templates to an OCI registry.

```bash
# Publish all features under src/
devcontainer features publish \
  -r ghcr.io \
  -n myorg \
  ./src

# Publish all templates under src/
devcontainer templates publish \
  -r ghcr.io \
  -n myorg \
  ./src
```

Requires `GITHUB_TOKEN` or equivalent registry credential in env.

## Common Workflows

### Start fresh (destroy + rebuild)

```bash
devcontainer up --workspace-folder . --remove-existing-container
```

### Test config before opening in VS Code

```bash
devcontainer read-configuration --workspace-folder . && devcontainer up --workspace-folder .
```

### Run tests inside the container from CI

```bash
devcontainer up --workspace-folder .
devcontainer exec --workspace-folder . npm test
```

### Pre-build and cache for CI/Codespaces

```bash
devcontainer build \
  --workspace-folder . \
  --image-name ghcr.io/org/myproject-dev:latest \
  --cache-from ghcr.io/org/myproject-dev:latest \
  --push true
```

Then in devcontainer.json:

```json
{
  "image": "ghcr.io/org/myproject-dev:latest"
}
```
