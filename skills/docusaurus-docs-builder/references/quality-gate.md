# Quality Gate

Run this before claiming Docusaurus docs are done.

## Build

- Docusaurus build passes.
- Broken links are fixed or explicitly documented.
- Generated files are not accidentally committed unless intended.
- Routes match expected URLs.

## Navigation

- Sidebar is task-oriented.
- Homepage and docs landing page route users quickly.
- Search exists or the lack of search is intentional.
- Footer and navbar links are current.

## Content

- Quickstart reaches first success fast.
- Install commands match the repo package manager.
- Examples are realistic and copy-pastable.
- Reference pages are not mixed into conceptual guides.
- Troubleshooting covers likely first-run failures.

## Visual Quality

- Pages are not walls of text.
- Diagrams, cards, tabs, and callouts clarify real content.
- Screenshots and images load and are readable.
- Brand colors work in light/dark mode.
- Mobile layout is usable.

## Claims

- No fake metrics.
- No unsupported performance or compatibility claims.
- Roadmap is separated from current functionality.
- Screenshots represent real behavior.

## Final Report

State:

- pages changed or created
- components changed or created
- build command and result
- links/images verified
- remaining live-browser checks or asset work
